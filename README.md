### Your first mission

Welcome to the team! Before we start building complex apps and robots, we need to set up our communication lines.

In the software world, we don't email code to each other. We use **Git** and **GitHub**. This repository is your "playground" to learn the workflow we will use for our actual projects.

## ðŸŽ¯ The Objective
Your goal is simple: **Add your name to the `members` folder.**

By doing this, you will learn the three pillars of our workflow:
1.  **Forking:** Copying a project so you can work on it safely.
2.  **Committing:** Saving your changes.
3.  **Pull Request (PR):** Asking the team to accept your work.

## ðŸ›  Instructions (The Web Flow)
*You don't need to install anything on your laptop yet. You can do this entirely in your browser.*

### Step 1: Fork this Repository
Look at the top right corner of this page. Click the button that says **Fork**.
*(This creates a copy of this project in your own account).*

### Step 2: Create Your File
1.  In your new forked repo, look for the folder named `members`. Click it.
2.  Click **Add file** > **Create new file**.
3.  Name the file: `YourFirstName_YourLastName.md` (Example: `bensmaine_mohamed.md`).

### Step 3: Write Your Bio
Copy and paste the template below into your new file and fill it out:

```markdown
# [Your Name]
**Role:** Tech Dept Member
**Major:** [Your Major]

## âš¡ About Me
I am interested in [Web Dev / AI / Cyber Security].
My goal for this semester is to learn [Skill].

## ðŸ’» Tech Stack

```

### Step 4: Commit Changes
On the top right of the screen, click "Commit changes...". In the "Commit message" box, write: `Added [Your Name] to the list`. Then click **Commit changes**.

### Step 5: The Pull Request (Crucial!)
1.  Go back to the main page of your repository.
2.  You should see a banner saying "This branch is 1 commit ahead of...". Click **Contribute** > **Open Pull Request**.
3.  Click **Create Pull Request**.
